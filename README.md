Summary: 
Основное задание (обучение DeepCNN для CIFAR-10)
1.	Подготовка окружения – фиксация генераторов случайных чисел (воспроизводимость), определение устройства (GPU/CPU).
2.	Аугментация данных – создание преобразований для обучения (случайный кроп, отражение, изменение цвета, нормализация, Cutout) и для теста (только нормализация).
3.	Загрузка CIFAR-10 – скачивание датасета, разделение обучающей выборки на обучающую (90%) и валидационную (10%).
4.	Создание DataLoader’ов – формирование батчей размером 64, перемешивание для обучения, использование нескольких потоков.
5.	Построение модели DeepCNN – инициализация свёрточной сети с 4 блоками (Conv → BN → ReLU → Conv → BN → ReLU → MaxPool → Dropout2d) и классификатором (AdaptiveAvgPool → Flatten → Linear).
6.	Компиляция – выбор функции потерь (CrossEntropyLoss), оптимизатора AdamW и планировщика скорости обучения (косинусное затухание с прогревом).
7.	Цикл обучения (до 100 эпох):
• Для каждой эпохи: обучаем на тренировочных батчах (вычисляем потери, градиенты, обновляем веса), оцениваем на валидационной выборке.
•	Обновляем learning rate через планировщик.
•	Если валидационная точность улучшилась – сохраняем модель.
•	Если точность не улучшается 20 эпох подряд – останавливаем обучение (early stopping).
8.	Оценка на тестовой выборке – загрузка лучшей модели, расчёт точности и потерь на тестовых данных.
9.	Визуализация – построение графиков точности и потерь для тренировочной и валидационной выборок, сохранение графиков.

Дополнительное задание (эксперименты с dropout)
1.	Подготовка данных – аналогично основному заданию (аугментация, загрузка, разбиение, DataLoader’ы).
2.	Задание списка dropout_rate для экспериментов – например, [0.0, 0.3, 0.5] (в коде так).
3.	Для каждого значения dropout_rate:
o	Создаём модель DeepCNN с указанным dropout_rate (в свёрточных блоках) и фиксированным Dropout(0.3) в классификаторе.
o	Настраиваем оптимизатор AdamW и планировщик косинусного затухания с прогревом (5 эпох).
o	Запускаем сокращённое обучение (максимум 50 эпох) с early stopping (patience=10).
o	Отслеживаем лучшую валидационную точность, запоминаем веса.
o	После завершения обучения (или остановки) загружаем лучшие веса и вычисляем точность на тестовой выборке.
o	Запоминаем результат для текущего dropout_rate.
4.	Сравнение результатов – выводим точности для каждого dropout_rate, определяем лучший параметр.
5.	Вывод – сообщаем, улучшил ли dropout качество по сравнению с базовой моделью (dropout=0.0) и на сколько.
